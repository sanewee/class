두더지 굴 — DFS(재귀) vs BFS(큐) 구조 비교

스택/큐에 담긴 칸을 격자에 직접 표시 → 구조의 모양 차이를 관찰

속도

굴 칸을 클릭하면 그 칸에서 시작. DFS 스택 / BFS 큐에 지금 들어있는 칸을 진하게 표시. 옅은 칸은 이미 처리 완료.

DFS 재귀 · 호출 스택

진한 칸이 가느다란 한 줄 → 깊이 파고든 경로

호출 스택 깊이0
한 갈래로 끝까지 → 막히면 되돌아옴(백트랙). 스택이 길고 얇음.
void dfs(int a, int b, int c){
  A[a][b] = c;                // 방문 표시
  for(int i=0; i<4; i++)
    if(safe(a+dx[i],b+dy[i]) && A[a+dx[i]][b+dy[i]]==1)
      dfs(a+dx[i], b+dy[i], c); // 재귀 호출
}

BFS 반복문 · 큐

진한 칸이 넓은 띠 → 같은 거리의 탐색 앞면

큐 크기0
가까운 칸부터 층층이 퍼짐. 큐가 넓게 벌어짐.
void bfs(int a, int b, int c){
  queue<VERTEX> Q;
  Q.push({a,b}), A[a][b]=c;    // 시작칸 넣고 방문
  while(!Q.empty()){          // 큐가 빌 때까지
    VERTEX cur = Q.front(); Q.pop();
    for(int i=0; i<4; i++)
      if(safe(..) && A[..]==1){
        A[..]=c; Q.push({..}); // 이웃 추가
      }
  }
}
관찰 포인트 — 진하게 표시된 칸의 모양을 봐. DFS는 늘 가느다란 실선, BFS는 두툼한 띠.
이게 "깊게 vs 넓게"의 실체. 스택은 한 경로만 기억하고, 큐는 앞면 전체를 기억함.
